Allgemein

Die gesamte interne Datenbank der App wird in eine Datei pro Tabelle exportiert. Dadurch sind die z.B. die Daten einer Tankfüllung in mehreren Dateien verteilt. Um das dazugehörige Auto einer Tankfüllung zu finden, suchen Sie den Wert der car Spalte in der Id Spalte der cars Tabelle.

Spalten

car

_id: [int]
car__name: [string]
color: [int] In der Android Dokumentation für Color finden Sie detaillierte Informationen darüber, wie der Farbwert berechnet wird.
initial_mileage: [int] Initialer Kilometerstand, wenn Sie anfangen das Auto mit der App zu benutzen.
suspended_since: [date]
buying_price: [float] Kaufpreis des Fahrzeugs.
num_tires: [int] Anzahl der Reifen des Fahrzeugs.

fuel_type

_id: [int]
fuel_type__name: [string]
category: [string]

other_cost

_id: [int]
car_id: [int] die Id des Autos.
mileage: [int] der Gesamtkilometerstand.
price: [float]
recurrence_interval: [int] 0 = einmalig, 1 = täglich, 2 = monatlich, 3 = jedes Quartal, 4 = jährlich
recurrence_multiplier: [int] Gibt zusammen mit recurrence_interval den Wiederholungszyklus an. Beispiel: interval = 2 und multiplier = 5 bedeutet jeden 5. Monat.
title: [string]
date: [date]
note: [string]
end_date: [date] Beende die Wiederholungen an diesem Datum.

refueling

_id: [int]
partial: [bool] (1 oder 0)
date: [date]
note: [string]
volume: [float]
fuel_type_id: [int] die Id der Spritart.
station_id: [int] die Id der Tankstelle.
price: [float]
mileage: [int] der Gesamtkilometerstand.
car_id: [int] die Id des Autos.

reminder

_id: [int]
title: [string]
after_time_span_unit: [int] 0 = Tage, 1 = Monate, 2 = Jahre
after_time_span_count: [int] Gibt zusammen mit after_time_span_unit eine Zeitspanne an. Beispiel: unit = 1 und count = 5 bedeutet nach 5 Monaten.
after_distance: [int]
start_date: [date]
start_mileage: [int]
car_id: [int] die Id des Autos.
notification_dismissed: [bool] (1 oder 0)
snoozed_until: [date]

station

_id: [int]
station__name: [string] Name der Tankstelle.

trip

_id: [int]
car_id: [int] die Id des Autos.
refueling_id: [int] die Id der verknüpften Tankfüllung.
date: [date] Datum bei Abfahrt.
date_end: [date] Datum bei Ankunft (für Fahrten über Nacht).
time_start: [time]
time_end: [time]
route_target: [string] Beschreibung der Strecke oder des Ziels.
purpose: [string] Zweck der Fahrt (z. B. geschäftlich, privat).
companies_visited: [string]
driver: [string]
occupants: [int]
cargo: [string]
km_start: [int] Kilometerstand bei Abfahrt.
km_end: [int] Kilometerstand bei Ankunft.
km_business: [int] Geschäftlich gefahrene Kilometer.
km_private: [int] Privat gefahrene Kilometer.
km_home_work: [int] Kilometer zwischen Wohnung und Arbeitsstätte.
fuel_liters: [float]
fuel_cost: [float]
other_costs_description: [string]
other_costs_amount: [float]
created_at: [datetime]
updated_at: [datetime]

trip_prefab

_id: [int]
car_id: [int] die Id des Autos.
type: [string] Typ des vordefinierten Wertes (route, purpose, driver, company).
value: [string]
usage_count: [int] Wie oft dieser Wert verwendet wurde.

tire_list

_id: [int]
buy_date: [date]
trash_date: [date] Datum, an dem die Reifen entsorgt wurden.
price: [float]
quantity: [int]
manufacturer: [string]
model: [string]
note: [string]
car_id: [int] die Id des Autos.

tire_usage

_id: [int]
distance_mount: [int] Kilometerstand bei Montage.
date_mount: [date]
distance_umount: [int] Kilometerstand bei Demontage.
date_umount: [date]
tire_id: [int] die Id des Reifens aus der Tabelle tire_list.

Fahrten Import/Export

Zusätzlich zur globalen Datensicherung ermöglicht das Fahrtenbuch-Modul das unabhängige Importieren und Exportieren von Fahrten. Dies ist nützlich für die Migration von Daten aus anderen Apps oder zur Erfüllung externer Berichtspflichten.

Die Anwendung unterstützt mehrere Formate, darunter ein generisches komma-separiertes Format und spezifische Formate wie Skoda (Semikolon). Beim Import versucht die App, die Spalten basierend auf den CSV-Headern zuzuordnen.
